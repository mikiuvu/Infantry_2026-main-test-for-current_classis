# 视觉通信模块接口说明

## 📚 模块概述

视觉通信模块（master_process）负责机器人主控板与上位机视觉程序之间的数据交互，支持自瞄、能量机关等功能。

## 🔌 通信方式

### 1. 串口模式 (VISION_USE_UART)
- **协议**: seasky协议
- **硬件**: USART1/USART6等
- **特点**: 传输可靠，适合长距离
- **配置**: 在 `robot_def.h` 中定义 `VISION_USE_UART`

### 2. USB虚拟串口模式 (VISION_USE_VCP)
- **协议**: 简单协议（直接传输结构体）
- **硬件**: USB接口
- **特点**: 速度快(12Mbps)，调试方便
- **配置**: 在 `robot_def.h` 中定义 `VISION_USE_VCP`

## 📊 数据结构

### 发送数据 (Vision_Send_s)

```c
typedef struct {
    uint16_t head;              // 包头 0xA5
    uint8_t mode;               // 工作模式
    Detect_Color_e foeColor;    // 敌方颜色
    float robotYaw;             // 机器人Yaw角度(°)
    float robotPitch;           // 机器人Pitch角度(°)
    float muzzleSpeed;          // 弹丸速度(m/s)
} Vision_Send_s;
```

**大小**: 16字节

| 字段 | 类型 | 大小 | 说明 |
|------|------|------|------|
| head | uint16_t | 2字节 | 包头标识(0xA5) |
| mode | uint8_t | 1字节 | 工作模式(自瞄/小符/大符) |
| foeColor | enum | 1字节 | 敌方颜色(0:蓝/1:红) |
| robotYaw | float | 4字节 | 云台航向角 |
| robotPitch | float | 4字节 | 云台俯仰角 |
| muzzleSpeed | float | 4字节 | 弹丸速度 |

### 接收数据 (Vision_Recv_s)

```c
typedef struct {
    uint16_t head;      // 包头 0xA5
    uint8_t fire;       // 开火标志
    uint8_t tracking;   // 跟踪标志
    float aimYaw;       // 目标Yaw角度(°)
    float aimPitch;     // 目标Pitch角度(°)
} Vision_Recv_s;
```

**大小**: 12字节

| 字段 | 类型 | 大小 | 说明 |
|------|------|------|------|
| head | uint16_t | 2字节 | 包头标识(0xA5) |
| fire | uint8_t | 1字节 | 开火标志(0:不开火/1:开火) |
| tracking | uint8_t | 1字节 | 跟踪标志(0:未跟踪/1:跟踪中) |
| aimYaw | float | 4字节 | 目标航向角 |
| aimPitch | float | 4字节 | 目标俯仰角 |

## 🔧 API接口

### 1. VisionInit()

**功能**: 初始化视觉通信模块

```c
Vision_Recv_s *VisionInit(UART_HandleTypeDef *_handle);
```

**参数**:
- `_handle`: 串口句柄指针(UART模式有效，VCP模式传NULL)

**返回值**:
- 返回接收数据结构体指针，用于读取视觉返回的数据

**使用示例**:
```c
// 在robot_cmd.c或gimbal.c的初始化函数中调用
Vision_Recv_s *vision_data = VisionInit(&huart1);
```

### 2. VisionSetFlag()

**功能**: 设置视觉识别参数

```c
void VisionSetFlag(Detect_Color_e detect_color, 
                   Work_Mode_e work_mode, 
                   Bullet_Speed_e bullet_speed);
```

**参数**:
- `detect_color`: 敌方颜色
  - `COLOR_RED`: 识别红色装甲板
  - `COLOR_BLUE`: 识别蓝色装甲板
  
- `work_mode`: 工作模式
  - `VISION_MODE_AIM`: 自瞄模式
  - `VISION_MODE_SMALL_BUFF`: 小能量机关
  - `VISION_MODE_BIG_BUFF`: 大能量机关
  
- `bullet_speed`: 弹丸速度
  - `SMALL_AMU_15`: 15m/s (步兵标准)
  - `SMALL_AMU_18`: 18m/s (步兵高速)
  - `BIG_AMU_16`: 16m/s (英雄)
  - `SMALL_AMU_30`: 30m/s (哨兵)

**使用示例**:
```c
// 根据裁判系统设置敌方颜色
Detect_Color_e enemy_color = (referee_data->self_color == COLOR_RED) 
                               ? COLOR_BLUE : COLOR_RED;

// 设置为自瞄模式，弹速15m/s
VisionSetFlag(enemy_color, VISION_MODE_AIM, SMALL_AMU_15);
```

### 3. VisionSetAltitude()

**功能**: 设置机器人姿态角度

```c
void VisionSetAltitude(float yaw, float pitch, float roll);
```

**参数**:
- `yaw`: Yaw轴角度(航向角，单位:度)
- `pitch`: Pitch轴角度(俯仰角，单位:度)
- `roll`: Roll轴角度(翻滚角，当前未使用，传0)

**使用示例**:
```c
// 在云台任务中实时更新
VisionSetAltitude(gimbal_imu_data->yaw, 
                  gimbal_imu_data->pitch, 
                  0);
```

### 4. VisionSend()

**功能**: 发送数据给视觉

```c
void VisionSend();
```

**使用示例**:
```c
// 在云台任务中周期性调用(建议1kHz)
void GimbalTask() {
    // 更新姿态
    VisionSetAltitude(yaw, pitch, 0);
    
    // 更新参数
    VisionSetFlag(enemy_color, work_mode, bullet_speed);
    
    // 发送数据
    VisionSend();
    
    // ... 其他控制逻辑
}
```

## 📝 使用流程

### 完整使用示例

```c
// ==================== 1. 初始化 ====================
// 在robot_cmd.c的RobotCMDInit()中
Vision_Recv_s *vision_recv_data;

void RobotCMDInit() {
    // 初始化视觉通信(UART1)
    vision_recv_data = VisionInit(&huart1);
    // ... 其他初始化
}

// ==================== 2. 实时发送 ====================
// 在robot_cmd.c的RobotCMDTask()中
void RobotCMDTask() {
    // 获取当前姿态(来自INS)
    float current_yaw = ins_data->yaw;
    float current_pitch = ins_data->pitch;
    
    // 获取敌方颜色(来自裁判系统)
    Detect_Color_e enemy_color = (referee_data->robot_id > 7) 
                                  ? COLOR_RED : COLOR_BLUE;
    
    // 获取弹速(来自裁判系统或配置)
    Bullet_Speed_e bullet_speed = SMALL_AMU_15;
    
    // 设置姿态
    VisionSetAltitude(current_yaw, current_pitch, 0);
    
    // 设置参数
    VisionSetFlag(enemy_color, VISION_MODE_AIM, bullet_speed);
    
    // 发送数据
    VisionSend();
}

// ==================== 3. 读取数据 ====================
// 在gimbal.c的GimbalTask()中使用视觉数据
void GimbalTask() {
    // 检查是否开启视觉模式
    if (gimbal_cmd_recv.gimbal_mode == GIMBAL_VISION_MODE) {
        // 检查是否正在跟踪目标
        if (vision_recv_data->tracking) {
            // 使用视觉返回的目标角度
            float target_yaw = vision_recv_data->aimYaw;
            float target_pitch = vision_recv_data->aimPitch;
            
            // 设置云台目标角度
            gimbal_yaw_motor->pid_ref = target_yaw;
            gimbal_pitch_motor->pid_ref = target_pitch;
        }
        
        // 检查是否允许开火
        if (vision_recv_data->fire && shoot_ready) {
            // 触发开火
            ShootControl(SHOOT_FIRE);
        }
    }
}
```

## ⚙️ 枚举类型说明

### 开火模式 (Fire_Mode_e)
```c
NO_FIRE = 0      // 不开火
AUTO_FIRE = 1    // 自动开火
AUTO_AIM = 2     // 自瞄辅助
```

### 目标状态 (Target_State_e)
```c
NO_TARGET = 0          // 未发现目标
TARGET_CONVERGING = 1  // 目标收敛中
READY_TO_FIRE = 2      // 准备开火
```

### 目标类型 (Target_Type_e)
```c
NO_TARGET_NUM = 0  // 无目标
HERO1 = 1          // 英雄(1号)
ENGINEER2 = 2      // 工程(2号)
INFANTRY3 = 3      // 步兵(3号)
INFANTRY4 = 4      // 步兵(4号)
INFANTRY5 = 5      // 步兵(5号)
OUTPOST = 6        // 前哨站
SENTRY = 7         // 哨兵(7号)
BASE = 8           // 基地
```

## 🐛 调试技巧

### 1. 检查通信是否正常
```c

if (DaemonIsOnline(vision_daemon_instance)) {
    // 视觉在线
} else {
    // 视觉离线，检查连接
}
```

        vision_recv_data->fire,
        vision_recv_data->tracking,
        vision_recv_data->aimYaw,

```c
LOGINFO("Send: color=%d, yaw=%.2f, pitch=%.2f, speed=%.1f",
        send_data.foeColor,
        send_data.robotYaw,
        send_data.robotPitch,
        send_data.muzzleSpeed);
```

## ⚠️ 注意事项

### 1. 数据对齐
- 使用 `#pragma pack(1)` 确保1字节对齐
- 不同编译器可能有不同的默认对齐方式

### 2. 坐标系约定
- Yaw角度定义必须与视觉程序一致
- Pitch角度正负方向需要统一
- 建议使用右手系

### 3. 弹速精度
- 弹速误差会直接影响自瞄精度
- 建议定期测量实际弹速
- 不同电压下弹速可能有变化

### 4. 通信频率
- 建议发送频率: 500Hz ~ 1kHz
- 频率过低: 目标跟踪延迟大
- 频率过高: 占用过多带宽

### 5. 离线处理
- 视觉离线时应切换到手动模式
- daemon会自动检测并尝试恢复通信
- 超过reload_count次未收到数据判定为离线
