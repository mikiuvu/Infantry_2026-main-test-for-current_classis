一、引言 
在赛场上裁判系统会对底盘一路的功率进行限制，但是目前大多电机并不能直接反馈功率，但是能够反
馈电流和转速；
那么，是什么导致了电机功率上升呢？一是对外做功，二是电流的一些‘’内耗‘，当然，电流也是正常工作
的电机能够对外做功的前提，因此，我们可以考虑利用电流和电机转速来拟合
二、思路 
如果你学过科学计算与仿真这门课，那么你一定了解过：对于一段曲线，你可以用与其相关的自变量x做
一个拟合，在一个区间内获得一个y关于x的函数，并且在这个区间内他是相对可靠的
因此我们可以建立一个数学模型P(I, )=K0+K1*I +K2* +K3*I* +K4* +K5*
（这里的K0）其实有点像直流分量的意味，其他两个变量让功率“骑”这个分量上波动，实际上就是电机
不堵转静止的情况下的 约等于K0；而I采用的单位是安培， 采用的单位是rad/s，均采用自国际制单
位
接下来了解完了原理，只要将每个不同工况下的电机数据放进能够拟合数据的函数中即可，常用的可以
选择matlab或者python
三、参数辨识实践 
本次采用的是python来进行参数辨识
首先先用功率计能够读出电流和功率；还缺少了一个数据：
那么，可以选择将功率计与电机控制板通讯，将I、 、P都给打包，再通过USB2TTL发送给上位机，最
好是能够使用VOFA+、serial plot这一类的串口绘图软件，因为他们能够将一段时间内的历史数据导
出，这样才方便我们对数据进行拟合（在实际的实践中，我选择了单独拿了一块A板去做数据的采集而
减小我要测试这一功能对原来系统的影响，我不希望他会影响到我原本的接线之类）
在能够正常控制电机的情况下，我们只需要在串口的接收中断注册和功率计的通信，然后确认能够进入
中断即通信没有问题后
开始debug查看反馈数据是否无误（注意不同功率计的通信格式，中科大开源功率计的帧头尾各一位，
也就是中间12位数据为实际所需的物理量，然后索引1到4表示电流，5到8表示电压，9到12表示功率，
实际上我们只需要电流与功率）
不过注意，我们需要事先确认好电机反馈电流与实际电流的映射关系，拿3508为例，他需要乘上转换的
系数，然后观测电机反馈电流乘上这个系数后的值是否与功率计反馈电流十分接近，因为在最后我们预
测功率时采用的就是电机反馈电流
float current = motor->measure.real_current*20.0f/16384.0f;
接着我们就以一定的频率控制电机与发送电机的转速，同时将原先功率计的数据一同打包发出，导出数
据文件后就可以开始拟合
对于电机的转速，我们可以有以下选择:
一、正反转的u(t- )，也就是一个在某个时间点给定的阶跃的控制信号 
二、正反转的Asin(t)，一个正弦形式的控制信号 
三、正反转的at+b，也就是一个关于时间的一次函数控制信号（同理也可以有n次方，不多例举) 
四、可以自己再探索探索，这些是为了模拟不同的工况来拟合更好的参数 
尽量选择到自己需要的转速区间，在此为中心以一个合适但不小的的值拓延，然后以一定频率发送（可
以在串口接收中断里发送，也就是作为转发，中科大开源的功率计是1khz）
而后将数据通过上位机软件导出，进行拟合得到参数

{

    }
}

一、 
在实际预测模型中，电机的转速是有正负的，但是对于功率来说，正负并不影响，所以在建立模型时，
最好可以考虑用速度的绝对值来建模
但是在实际工作时，正负影响不大的原因在于首先电机转速或许没那么高，其次系数很小，但是最好考
虑用绝对值更准确
