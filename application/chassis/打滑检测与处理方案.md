# 底盘打滑检测与处理方案

## 一、打滑检测方法

### 1. 速度残差检测（最常用）

**原理**：比较轮速推算的速度与IMU/里程计估计的速度

```c
残差 = |轮速估计的速度| - |IMU估计的速度|
打滑 = 残差 > 阈值
```

| 优点 | 缺点 |
|------|------|
| 简单直接 | 阈值需要调参 |
| 计算量小 | 突变检测有延迟 |

---

### 2. 电机电流异常检测

**原理**：正常情况下高扭矩对应高加速度，打滑时电流大但速度不变

```c
if (motor_current > HIGH_THRESHOLD && speed_change < LOW_THRESHOLD) {
    slip_detected = true;
}
```

| 优点 | 缺点 |
|------|------|
| 响应快 | 需要电流反馈 |
| 不需要额外传感器 | 需区分堵转和打滑 |

---

### 3. 卡尔曼滤波残差检测（rm-controls方案）

**原理**：利用卡尔曼滤波的"创新量(Innovation)"判断观测异常

```
创新量 = 观测值 - 预测值
马氏距离 = Innovation^T * S^{-1} * Innovation
打滑 = 马氏距离 > χ² 阈值
```

**rm-controls**使用`robot_localization`包融合IMU和轮速里程计，当innovation过大时自动降权。

| 优点 | 缺点 |
|------|------|
| 理论完善，自动调整权重 | 实现复杂 |
| 与状态估计紧耦合 | 需要调整Q/R矩阵 |

---

### 4. 多轮一致性检测

**原理**：正常情况下4个轮子反推的底盘速度应一致，打滑轮会偏离

```c
float vx_from_lf = inverse_kinematics(wheel_lf, ...);
float vx_from_rf = inverse_kinematics(wheel_rf, ...);
float vx_from_lb = inverse_kinematics(wheel_lb, ...);
float vx_from_rb = inverse_kinematics(wheel_rb, ...);

float variance = calculate_variance(vx_from_lf, vx_from_rf, vx_from_lb, vx_from_rb);

if (variance > THRESHOLD) {
    slip_wheel = find_outlier_wheel();  // 定位打滑轮
}
```

| 优点 | 缺点 |
|------|------|
| 能定位哪个轮子打滑 | 计算量稍大 |
| 不需要IMU | 全部轮子同时打滑时失效 |

---

### 5. 加速度一致性检测

**原理**：轮速微分得到的加速度应与IMU加速度一致

```c
float wheel_accel = (wheel_vel - last_wheel_vel) / dt;
float imu_accel = Chassis_IMU_data->Accel[X];

if (fabs(wheel_accel - imu_accel) > THRESHOLD) {
    slip_detected = true;
}
```

| 优点 | 缺点 |
|------|------|
| 响应快（加速度是速度的导数） | 微分噪声大 |
| 直接反映动力学异常 | 需要滤波处理 |

---

## 二、打滑处理策略

### 1. 降低输出/限制加速度（最常用）

**原理**：打滑时减小输出力矩，让轮子恢复抓地力

```c
if (slip_detected) {
    motor_output *= SLIP_OUTPUT_RATIO;  // 例如0.5~0.7
}
```

**适用场景**：
- 起步加速过猛
- 地面湿滑
- 爬坡打滑

---

### 2. 牵引力控制（TCS - Traction Control System）

**原理**：类似汽车防滑系统，限制驱动轮的滑移率

```c
// 滑移率 = (轮速 - 车速) / 轮速
float slip_ratio = (wheel_speed - chassis_speed) / wheel_speed;

// 目标滑移率通常在10%~20%之间效率最高
if (slip_ratio > TARGET_SLIP_RATIO) {
    reduce_torque(motor);
}
```

| 滑移率 | 状态 |
|--------|------|
| 0% | 纯滚动，无驱动力传递 |
| 10-20% | 最佳抓地力区间 |
| 100% | 完全打滑，空转 |

---

### 3. 动态调整卡尔曼滤波参数

**原理**：打滑时增大轮速测量噪声R，降低轮速权重

```c
if (slip_detected) {
    kf_vx.R = 5.0f;   // 增大测量噪声（不信任轮速）
    kf_vy.R = 5.0f;
} else {
    kf_vx.R = 0.5f;   // 正常值
    kf_vy.R = 0.5f;
}
```

**效果**：打滑时更信任IMU积分，避免轮速污染状态估计

---

### 4. 单轮打滑处理

**原理**：定位到打滑轮后，单独降低该轮输出

```c
if (wheel_lf_slipping) {
    motor_lf_output *= 0.5f;
    // 其他轮保持正常，维持机动性
}
```

**优点**：不影响整体机动性，精准处理

---

### 5. 切换控制模式

**原理**：打滑严重时切换到保守模式

```c
if (severe_slip) {
    chassis_mode = CHASSIS_SLOW_MODE;
    max_speed *= 0.5f;
    max_accel *= 0.3f;
}
```

---

### 6. 前馈补偿

**原理**：打滑时增加前馈，帮助电机维持期望速度

```c
// 正常情况
output = PID_output;

// 打滑时增加前馈
if (slip_detected) {
    output = PID_output + feedforward * target_accel;
}
```

**注意**：这种方法要谨慎使用，可能加剧打滑

## 三、实战建议

### 推荐方案组合

| 场景 | 检测方法 | 处理策略 |
|------|----------|----------|
| **一般步兵** | 速度残差 + 卡尔曼融合 | 动态调整R + 限制输出 |
| **英雄（重载）** | 电流异常 + 多轮一致性 | TCS牵引力控制 |
| **哨兵（高速）** | 加速度一致性 | 降低加速度限制 |
| **平衡步兵** | 全部方法综合 | 紧急模式切换 |

### 参数调节建议

```c
// 打滑检测阈值（需根据实际调参）
#define SLIP_RATIO_THRESHOLD    0.3f    // 滑移率阈值 30%
#define RESIDUAL_THRESHOLD      500.0f  // 速度残差阈值 mm/s
#define CURRENT_THRESHOLD       8000    // 电流阈值 mA

// 打滑处理参数
#define SLIP_OUTPUT_RATIO       0.6f    // 打滑时输出系数
#define SLIP_R_INCREASE         10.0f   // 卡尔曼R增大倍数
#define RECOVERY_TIME           0.5f    // 恢复时间 s
```

## 四、当前框架实现（已完成）

### 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        底盘控制流程 (带打滑检测)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. EstimateSpeed() - 速度估计与打滑检测                                 │
│     ├─ 轮速逆运动学 → wheel_vx/vy                                       │
│     ├─ 卡尔曼融合(轮速+IMU) → real_vx/vy                                │
│     ├─ 滑移率计算 → slip_ratio                                          │
│     └─ TCS因子计算 → tcs_factor (打滑时降低)                            │
│                                                                         │
│  2. MecanumCalculate() - 正运动学解算                                    │
│     └─ chassis_vx/vy → vt_lf/rf/lb/rb                                   │
│                                                                         │
│  3. LimitChassisOutput() - 输出限制与闭环补偿                            │
│     ├─ 速度外环PID: (目标速度 - 融合速度) → 补偿量                       │
│     ├─ 打滑时: 补偿量 *= 0.3 (避免加剧打滑)                             │
│     └─ 最终输出 = (轮速 + 补偿) * power_buff * tcs_factor               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 关键参数

```c
// 打滑检测参数 (在chassis.c中定义)
#define SLIP_RATIO_THRESHOLD    0.25f   // 滑移率阈值 (超过25%认为打滑)
#define OPTIMAL_SLIP_RATIO      0.15f   // 最佳滑移率 (最大抓地力区间)
#define TCS_MIN_FACTOR          0.4f    // TCS最小输出系数
#define TCS_RECOVERY_RATE       0.02f   // TCS恢复速率 (每周期)
#define MIN_SPEED_FOR_SLIP      50.0f   // 最小速度阈值 (低于此不检测打滑)

// 速度外环PID参数
.Kp = 1.5f,
.Ki = 0.05f,
.Kd = 0.0f,
.MaxOut = 5000.0f,
```

### 工作流程

1. **正常状态**:
   - `tcs_factor = 1.0` → 全功率输出
   - 速度外环PID正常工作 → 提高响应速度
   - 卡尔曼R = 0.5 → 正常信任轮速

2. **打滑检测** (slip_ratio > 0.25):
   - `is_slipping = 1`
   - `tcs_factor` 降低 → 减小输出，节省功率
   - 速度外环补偿 *= 0.3 → 避免加剧打滑
   - 卡尔曼R = 5.0 → 降低轮速权重

3. **恢复阶段**:
   - `tcs_factor` 逐渐恢复 (+0.02/周期)
   - 平滑过渡，避免抖动

### Debug变量

```c
// SlipControl_t slip_ctrl 结构体包含:
slip_ctrl.slip_ratio_x   // X方向滑移率
slip_ctrl.slip_ratio_y   // Y方向滑移率
slip_ctrl.is_slipping    // 是否打滑 (0/1)
slip_ctrl.tcs_factor     // TCS输出系数 (0.4~1.0)
slip_ctrl.wheel_vx       // 轮速计算的vx
slip_ctrl.wheel_vy       // 轮速计算的vy

// 融合速度
chassis_feedback_data.real_vx
chassis_feedback_data.real_vy
```

### 调参建议

| 现象 | 调整方向 |
|------|----------|
| 打滑检测太灵敏 | 增大 `SLIP_RATIO_THRESHOLD` |
| 打滑检测太迟钝 | 减小 `SLIP_RATIO_THRESHOLD` |
| 打滑后恢复太慢 | 增大 `TCS_RECOVERY_RATE` |
| 打滑时输出太小 | 增大 `TCS_MIN_FACTOR` |
| 速度响应太慢 | 增大速度外环 `Kp` |
| 速度有振荡 | 减小速度外环 `Kp`，增加 `Kd` |
