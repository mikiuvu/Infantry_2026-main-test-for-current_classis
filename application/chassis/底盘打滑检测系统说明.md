# 底盘打滑检测系统说明

## 一、什么是打滑检测

打滑检测是指通过传感器数据判断底盘轮子是否与地面发生相对滑动的技术。

### 打滑的两种类型

| 类型 | 现象 | 原因 | 类比 |
|------|------|------|------|
| **驱动打滑** | 轮速 > 车速（轮子空转） | 加速过猛、地面湿滑 | 汽车TCS |
| **制动打滑** | 轮速 < 车速（轮子抱死） | 减速过猛 | 汽车ABS |

### 如何判断打滑？

核心思想：**比较轮子转速推算的速度与实际车体速度**

```
轮速推算速度 = 轮子角速度 × 轮子半径 / 减速比（再进行运动学逆解算）
实际车体速度 = 卡尔曼融合(IMU加速度 + 轮速)

滑移率 = |轮速推算速度 - 融合速度| / (|轮速推算速度| + 100)

滑移率 > 0.25 → 打滑
```

> **为什么分母加100？** 这是防止低速时误判的技巧：
> - 防止除零：避免轮速为0时分母为0
> - 低速容忍：低速时微小差异不会被放大成"打滑"

## 二、打滑检测的目的

### 1. 节省功率（最重要）

打滑时轮子空转，电机消耗大量电流但不产生有效推力：
- 功率被浪费在克服轮子惯性上
- 比赛中底盘功率受限，浪费=减少机动性

### 2. 提高控制精度

打滑时轮速反馈失真：
- 电机速度环认为"速度达标了"
- 实际底盘根本没动或没停住
- 检测打滑后可切换到IMU反馈

### 3. 保护机械结构

持续打滑导致：
- 轮子/辊子磨损加剧
- 电机过热
- 减速箱冲击

### 4. 提升操控体验

- 避免"踩油门不走"的无力感
- 避免"刹不住"滑出去的失控感

## 三、其他队伍的方案

### 方案1：速度残差检测（最常用）

```c
残差 = |轮速估计的速度| - |IMU估计的速度|
打滑 = 残差 > 阈值
```

**优点**：简单直接，计算量小  
**缺点**：阈值需要调参，突变检测有延迟

### 方案2：电机电流异常检测

```c
if (motor_current > HIGH && speed_change < LOW) {
    slip_detected = true;  // 高电流但速度不变 = 打滑
}
```

**优点**：响应快，不需要额外传感器  
**缺点**：需要电流反馈，需区分堵转和打滑

### 方案3：卡尔曼滤波残差检测（rm-controls）

使用`robot_localization`包融合IMU和轮速里程计，当卡尔曼滤波的"创新量(Innovation)"过大时判定打滑。

**优点**：理论完善，自动调整权重  
**缺点**：实现复杂，需要调整Q/R矩阵

### 方案4：多轮一致性检测

```c
// 4个轮子反推的底盘速度应该一致
float variance = Var(vx_from_lf, vx_from_rf, vx_from_lb, vx_from_rb);
if (variance > THRESHOLD) {
    slip_wheel = find_outlier();  // 定位打滑轮
}
```

**优点**：能定位哪个轮子打滑  
**缺点**：全部轮子同时打滑时失效

## 四、我的实现

### 整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                      底盘控制流程（带打滑检测）                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ EstimateSpeed() - 速度估计与打滑检测                         │   │
│  │  1. 轮速逆运动学 → wheel_vx/vy                              │   │
│  │  2. 卡尔曼融合(轮速+IMU) → real_vx/vy                       │   │
│  │  3. 滑移率计算 → slip_ratio                                 │   │
│  │  4. TCS因子计算 → tcs_factor                                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ MecanumCalculate() - 正运动学解算                            │   │
│  │  chassis_vx/vy → vt_lf/rf/lb/rb                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ LimitChassisOutput() - 输出控制                              │   │
│  │  1. 速度外环PID: (目标 - 融合速度) → 补偿量                  │   │
│  │  2. 打滑时: 补偿 *= 0.3, 输出 *= tcs_factor                 │   │
│  │  3. 最终输出 = (轮速 + 补偿) * power * tcs                  │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心代码结构

```c
/* ===================== 打滑检测与牵引力控制 ===================== */
typedef struct {
    float slip_ratio_x;      // X方向滑移率
    float slip_ratio_y;      // Y方向滑移率
    uint8_t is_slipping;     // 是否打滑
    float tcs_factor;        // TCS输出系数 (0.4~1.0)
    float wheel_vx;          // 轮速计算的vx
    float wheel_vy;          // 轮速计算的vy
} SlipControl_t;

// 打滑检测参数
#define SLIP_RATIO_THRESHOLD    0.25f   // 滑移率阈值
#define OPTIMAL_SLIP_RATIO      0.15f   // 最佳滑移率
#define TCS_MIN_FACTOR          0.4f    // TCS最小输出系数
#define TCS_RECOVERY_RATE       0.02f   // TCS恢复速率
```

### 速度融合（卡尔曼滤波）

```c
// 一维卡尔曼滤波器
// 预测: 用IMU加速度积分
// 校正: 用轮速逆运动学结果

x_pred = x + accel * dt;           // 预测
K = P / (P + R);                   // 卡尔曼增益
x = x_pred + K * (wheel_vel - x_pred);  // 更新

// 打滑时增大R，降低轮速权重
if (slipping) R = 5.0f; else R = 0.5f;
```

### 打滑检测逻辑

```c
// 计算滑移率
slip_ratio = |wheel_speed - fused_speed| / (|wheel_speed| + 100)

if (slip_ratio > 0.25) {
    is_slipping = 1;
    tcs_factor = max(0.4, 1.0 - (slip_ratio - 0.15) * 1.5);
} else {
    is_slipping = 0;
    tcs_factor 逐渐恢复到 1.0;
}
```

### 输出控制

```c
// 速度外环PID补偿
vx_compensation = PID(chassis_vx - real_vx);
vy_compensation = PID(chassis_vy - real_vy);

// 打滑时减弱补偿
if (slipping) compensation *= 0.3;

// 应用TCS因子
motor_output = (wheel_ref + compensation) * power_buff * tcs_factor;
```

## 五、为什么这么干

### 为什么用卡尔曼融合而不是简单平均？

| 方法 | 问题 |
|------|------|
| 只用轮速 | 打滑时完全失真 |
| 只用IMU积分 | 长时间漂移严重 |
| 简单加权平均 | 权重固定，无法适应打滑 |
| **卡尔曼融合** | 自动调整权重，打滑时信任IMU |

### 为什么打滑时要降低输出而不是增加？

**物理原理**：
- 轮子与地面的摩擦力分为静摩擦和滑动摩擦
- 滑动摩擦力 < 静摩擦力
- 打滑时增加力矩 → 滑动更严重 → 有效推力更小 → 浪费功率

**正确做法**：
- 降低力矩 → 轮子恢复抓地 → 静摩擦恢复 → 有效推力恢复

### 为什么速度外环和TCS要结合？

| 单独使用 | 问题 |
|----------|------|
| 只有速度外环 | 打滑时拼命加输出，加剧打滑 |
| 只有TCS | 正常时响应变慢 |
| **结合使用** | 正常时外环提速，打滑时TCS保护 |

### 为什么打滑时外环补偿要减弱？

速度外环的目的是"让底盘速度跟上目标"。打滑时：
- 融合速度 < 目标速度 → 误差大 → 外环想增加输出
- 但增加输出会加剧打滑

所以打滑时：`补偿 *= 0.3`，让TCS主导控制。

## 六、这么干的好处

### 1. 功率效率提升

| 状态 | 传统方案 | 本方案 |
|------|----------|--------|
| 正常 | 100% | 100% |
| 打滑 | 浪费50%+ | 降低输出，恢复后再加速 |

### 2. 响应速度提升

速度外环用融合速度做反馈：
- 目标速度变化 → 立即计算补偿
- 不等轮速反馈，提前动作

### 3. 制动距离缩短

传统：轮子停了但车还在滑 → 电机不知道  
本方案：IMU感知到车还在动 → 继续反向输出

### 4. 自适应性

- 卡尔曼自动调整轮速/IMU权重
- TCS因子根据滑移率动态变化
- 无需针对不同地面手动调参

### 5. 调试友好

提供完整的Debug变量：
- 滑移率、TCS因子实时可观测
- 便于分析问题和调参

## 七、怎么使用我的打滑检测

### 前置条件

1. **双板模式** (`CHASSIS_BOARD` 宏定义)
2. 底盘板有独立IMU (BMI088)
3. 底盘电机使用M3508 + C620电调

### 参数调节

在 `chassis.c` 开头修改以下宏定义：

```c
// 打滑检测参数
#define SLIP_RATIO_THRESHOLD    0.25f   // 滑移率阈值
// ↑ 增大 = 不敏感，减小 = 敏感

#define TCS_MIN_FACTOR          0.4f    // TCS最小输出
// ↑ 增大 = 打滑时保留更多输出

#define TCS_RECOVERY_RATE       0.02f   // TCS恢复速率
// ↑ 增大 = 恢复更快
```

速度外环PID在 `ChassisInit()` 中：

```c
PID_Init_Config_s chassis_vel_pid_conf = {
    .Kp = 1.5f,      // 增大 = 响应更快，但可能振荡
    .Ki = 0.05f,     // 消除稳态误差
    .Kd = 0.0f,      // 抑制振荡
    .MaxOut = 5000.0f,
};
```

### Debug方法

使用Ozone/JLink RTT观察以下变量：

| 变量 | 含义 | 正常值 | 打滑时 |
|------|------|--------|--------|
| `slip_ctrl.slip_ratio_x` | X滑移率 | < 0.2 | > 0.25 |
| `slip_ctrl.is_slipping` | 打滑标志 | 0 | 1 |
| `slip_ctrl.tcs_factor` | TCS系数 | 1.0 | 0.4~0.9 |
| `kf_vx.x` | 融合速度X | 接近轮速 | 偏离轮速 |
| `chassis_feedback_data.real_vx` | 反馈速度 | - | - |

### 调试步骤

1. **确认IMU正常**
   - 检查 `Chassis_IMU_data->Accel[0/1]` 有数据
   - 静止时加速度接近 (0, 0, 9.8)

2. **确认轮速正常**
   - 检查 `motor_lf->measure.speed_aps` 等有反馈
   - 空转时方向和大小正确

3. **测试打滑检测**
   - 将底盘架起来（轮子悬空）
   - 给速度命令，观察 `is_slipping` 是否为1
   - 轮子接触地面后应恢复为0

4. **调整参数**
   - 如果太灵敏：增大 `SLIP_RATIO_THRESHOLD`
   - 如果不灵敏：减小阈值
   - 如果恢复太慢：增大 `TCS_RECOVERY_RATE`

### 禁用打滑检测

如果需要临时禁用，在 `LimitChassisOutput()` 中：

```c
float tcs = 1.0f;  // 强制为1，禁用TCS
// float tcs = slip_ctrl.tcs_factor;  // 原代码
```
