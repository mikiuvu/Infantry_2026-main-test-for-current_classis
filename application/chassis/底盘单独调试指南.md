# 底盘单独调试指南

## 一、配置检查清单

### 1. 板子类型定义 (`application/robot_def.h`)
确保定义为单板模式：
```c
#define ONE_BOARD // 单板控制整车 ✅
// #define CHASSIS_BOARD //底盘板
// #define GIMBAL_BOARD  //云台板
```

### 2. 硬件连接确认
- ✅ **必需硬件**：
  - [x] 4个底盘电机（M3508）连接到 CAN2，ID分别为：1, 2, 3, 4
  - [x] 遥控器接收机连接到 USART3（huart3）
  - [x] 调试器（Dap-Link/J-Link/ST-Link）

- ⚠️ **可选硬件**（如果没有可以注释掉初始化代码）：
  - [ ] 裁判系统连接到 USART6（huart6）
  - [ ] 超级电容连接到 CAN1

### 3. 可选模块处理

如果**没有裁判系统**，在 `chassis.c` 的 `ChassisInit()` 中注释掉：
```c
// referee_data = Referee_Interactive_init(&huart6,&ui_data); 
```
并在 `LimitChassisOutput()` 中注释掉相关代码：
```c
// msg.RefereePowerLimit = referee_data->GameRobotState.chassis_power_limit;
// msg.RefereeEnergyBuffer = referee_data->PowerHeatData.chassis_power_buffer;
```

如果**没有超级电容**，在 `chassis.c` 的 `ChassisInit()` 中注释掉：
```c
// SuperCap_Init_Config_s cap_conf = {...};
// cap = SuperCapInit(&cap_conf);
```
并在 `LimitChassisOutput()` 中注释掉：
```c
// SuperCapSend(cap,(uint8_t*)&msg);
```

## 二、电机ID配置

在 `application/chassis/chassis.c` 的 `ChassisInit()` 函数中（约102-116行）：

```c
chassis_motor_config.can_init_config.tx_id = 3;  // 左前轮
chassis_motor_config.can_init_config.tx_id = 4;  // 右前轮
chassis_motor_config.can_init_config.tx_id = 2;  // 左后轮
chassis_motor_config.can_init_config.tx_id = 1;  // 右后轮
```

**电机正反转标志**：
- 左前轮(lf)：`MOTOR_DIRECTION_NORMAL`
- 右前轮(rf)：`MOTOR_DIRECTION_REVERSE`
- 左后轮(lb)：`MOTOR_DIRECTION_NORMAL`
- 右后轮(rb)：`MOTOR_DIRECTION_REVERSE`

## 三、PID参数调试

### 1. 速度环PID（第76-84行）
```c
.speed_PID = {
    .Kp = 4.5,  // 比例系数，影响响应速度
    .Ki = 0,    // 积分系数，消除稳态误差
    .Kd = 0,    // 微分系数，抑制超调
    .IntegralLimit = 3000,
    .MaxOut = 12000,
},
```

### 2. 电流环PID（第85-92行）
```c
.current_PID = {
    .Kp = 0.5,  // 电流环比例系数
    .Ki = 0,
    .Kd = 0,
    .IntegralLimit = 3000,
    .MaxOut = 15000,
},
```

**调参建议**：
1. 先调速度环：逐步增大Kp直到出现震荡，然后减小到震荡消失
2. 如有稳态误差，小幅增加Ki
3. 如有超调，适当增加Kd

## 四、底盘运动学解算

当前默认启用**方案A: X型全向轮解算**（第174-178行）

如需切换到其他方案：
- **麦克纳姆轮**：取消注释第168-171行
- **十字型全向轮**：取消注释第180-186行

## 五、遥控器控制说明

底盘通过 `robot_cmd` 模块接收遥控器指令，主要控制：

### 遥控器模式：
- 右摇杆：控制底盘前后左右移动（vx, vy）
- 左摇杆左右：控制底盘旋转（wz）

### 底盘控制模式（在robot_cmd中设置）：
- `CHASSIS_NO_FOLLOW`：底盘不旋转，只平移
- `CHASSIS_FOLLOW_GIMBAL_YAW`：底盘跟随云台方向
- `CHASSIS_ROTATE`：小陀螺模式

## 七、常见问题

### 1. 电机不转
- 检查CAN连接是否正常
- 检查电机ID是否正确
- 检查遥控器是否在线
- 检查是否进入急停模式

### 2. 电机转向错误
- 调整 `motor_reverse_flag` 为 `MOTOR_DIRECTION_REVERSE` 或 `MOTOR_DIRECTION_NORMAL`

### 3. 运动方向不对
- 检查运动学解算公式是否匹配实际轮子布局
- 检查坐标系定义（右手系，x前y右）

### 4. 编译错误
- 确保只定义了一个板子类型（ONE_BOARD/CHASSIS_BOARD/GIMBAL_BOARD）
- 检查是否所有依赖的模块都已初始化

## 八、调试流程建议

1. **单电机测试**：
   - 先注释掉其他3个电机的初始化
   - 只测试一个电机能否正常转动
   - 确认ID和方向正确

2. **逐步增加电机**：
   - 依次添加其他电机
   - 测试每个电机的转向

3. **运动学测试**：
   - 测试前进、后退
   - 测试左平移、右平移
   - 测试原地旋转
   - 测试斜向移动

4. **PID调参**：
   - 记录初始参数
   - 逐步调整，每次只改一个参数
   - 记录效果，找到最优参数

## 九、安全注意事项

⚠️ **调试前必读**：
1. 确保底盘已架空或有足够的活动空间
2. 手边准备急停开关或遥控器急停拨杆
3. 初次上电时电机速度设置较低进行测试
4. 有异常立即断电
5. 检查电机温度，过热需停止调试
