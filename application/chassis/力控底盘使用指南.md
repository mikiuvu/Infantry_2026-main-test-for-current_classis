# 力控底盘使用指南

## 快速开始

### 1. 编译与烧录

```powershell
# 编译
mingw32-make -j24

# 烧录 (DAP-Link)
mingw32-make download_dap

# 烧录 (J-Link)
mingw32-make download_jlink
```

### 2. 首次使用检查清单

- [ ] 确认 `robot_def.h` 中的板型定义 (`ONE_BOARD` 或 `CHASSIS_BOARD`)
- [ ] 确认轮子半径 `RADIUS_WHEEL` 和减速比 `REDUCTION_RATIO_WHEEL`
- [ ] 确认电机ID与实际接线一致
- [ ] 修改 `ROBOT_MASS` 为实际机器人质量

## 调参指南

### Step 1: 调电流环 (最内环)

**目标**: 电机能稳定跟踪电流参考值

```c
// chassis.c - ChassisInit()
.current_PID = {
    .Kp = 1.0f,   // 从1.0开始，观察电流跟踪
    .MaxOut = 16000,
},
```

**调参方法**:
1. 用VOFA+观察 `motor->measure.real_current` vs `motor->measure.target_current`
2. Kp过小: 跟踪慢，有稳态误差
3. Kp过大: 振荡、发热

**建议范围**: `Kp = 0.8 ~ 2.0`

---

### Step 2: 调速度外环 (应用层)

**目标**: 底盘速度能跟踪目标速度

```c
// chassis.c - ChassisInit() 中的 force_vel_pid_conf
.Kp = 50.0f,    // 速度到力的增益
.Ki = 5.0f,     // 消除稳态误差
.Kd = 2.0f,     // 抑制超调
.MaxOut = 150.0f,  // 最大力 (N)
```

**调参顺序**:
1. **先调Kp**: 从30开始，逐步增大到响应足够快
2. **再调Kd**: 如果有超调/振荡，增加Kd
3. **最后调Ki**: 如果有稳态误差，少量增加Ki

**调参方法**:
1. 用VOFA+观察 `chassis_vx` vs `chassis_feedback_data.real_vx`
2. 给一个阶跃速度指令，观察响应曲线

| 现象 | 调整 |
|------|------|
| 响应慢 | 增大Kp |
| 超调大 | 减小Kp 或 增大Kd |
| 振荡 | 减小Kp 和 Ki |
| 稳态误差 | 增大Ki |

**建议范围**:
- `Kp = 30 ~ 80`
- `Ki = 2 ~ 10`
- `Kd = 0 ~ 5`

---

### Step 3: 调旋转力矩PID

**目标**: 自旋和跟随云台时的角速度控制

```c
// chassis.c - ChassisInit() 中的 force_wz_pid_conf
.Kp = 30.0f,
.Ki = 2.0f,
.Kd = 1.0f,
.MaxOut = 100.0f,  // 最大旋转力矩 (N·m)
```

---

### Step 4: 调打滑检测参数

```c
// chassis.c 顶部宏定义
#define TORQUE_EFFICIENCY_THRESHOLD  0.5f   // 效率阈值
#define TCS_MIN_FACTOR               0.4f   // 最小输出系数
#define TCS_RECOVERY_RATE            0.02f  // 恢复速率
#define MIN_FORCE_FOR_SLIP           50.0f  // 检测阈值 (N)
#define ROBOT_MASS                   15.0f  // 机器人质量 (kg)
```

**调参建议**:
- `ROBOT_MASS`: 必须设为实际质量，否则打滑检测不准
- `TORQUE_EFFICIENCY_THRESHOLD`: 0.4~0.6，越小越不敏感
- `TCS_MIN_FACTOR`: 0.3~0.5，打滑时的最小输出
- `TCS_RECOVERY_RATE`: 0.01~0.05，越大恢复越快

## VOFA+ 调试

### 连接设置
1. 串口选择对应COM口
2. 波特率: 根据 `bsp_vofa.c` 设置
3. 协议: JustFloat

### 当前发送的数据

| 通道 | 变量 | 含义 |
|------|------|------|
| CH1 | `Pitch` | IMU Pitch角 |
| CH2 | `Yaw` | IMU Yaw角 |
| CH3 | `chassis_vx` | 目标X速度 |
| CH4 | `real_vx` | 实际X速度 (融合) |
| CH5 | `tcs_factor` | TCS系数 |

### 自定义调试数据

修改 `ChassisTask()` 末尾的 `VOFA()` 调用:

```c
// 示例: 观察力矩输出
VOFA(torque_lf, torque_rf, torque_lb, torque_rb, slip_ctrl.torque_efficiency);

// 示例: 观察速度跟踪
VOFA(chassis_vx, chassis_feedback_data.real_vx, 
     chassis_vy, chassis_feedback_data.real_vy, 0);
```

## 常见问题排查

### Q1: 底盘不动
**检查**:
1. 电机是否使能 (不在 `CHASSIS_ZERO_FORCE` 模式)
2. 电机ID是否正确
3. CAN线是否连接
4. 电流环Kp是否为0

### Q2: 底盘抖动/振荡
**原因**: PID参数过大
**解决**:
1. 减小速度外环Kp
2. 减小或去掉Ki
3. 检查电流环是否振荡

### Q3: 响应慢，跟不上
**原因**: PID参数过小
**解决**:
1. 增大速度外环Kp
2. 检查 `MaxOut` 是否限制了输出

### Q4: 打滑时不减速
**检查**:
1. `ROBOT_MASS` 是否正确
2. IMU数据是否正常
3. `MIN_FORCE_FOR_SLIP` 是否太大

### Q5: 一直在"打滑"状态
**原因**: 打滑检测过于敏感
**解决**:
1. 减小 `TORQUE_EFFICIENCY_THRESHOLD` (如0.4)
2. 检查IMU加速度数据是否异常

## 切换回速控模式

如果需要切换回原来的速控模式:

### 1. 修改电机配置

```c
// chassis.c - ChassisInit()
.controller_setting_init_config = {
    .outer_loop_type = SPEED_LOOP,              // 改回速度环
    .close_loop_type = SPEED_LOOP | CURRENT_LOOP,  // 速度+电流
},
.controller_param_init_config = {
    .speed_PID = {
        .Kp = 2.0f,
        .Ki = 0.0f,
        .Kd = 0.0f,
        .MaxOut = 12000,
    },
    .current_PID = {
        .Kp = 0.5f,
        .MaxOut = 15000,
    },
},
```

### 2. 修改解算函数调用

```c
// ChassisTask() 中
// 将 ForceCalculate() 改回 MecanumCalculate()
MecanumCalculate();  // 速控用这个
// ForceCalculate();  // 力控用这个
```

### 3. 修改输出设置

```c
// LimitChassisOutput() 中
// 使用 vt_lf, vt_rf... 而不是 torque_lf, torque_rf...
DJIMotorSetRef(motor_lf, vt_lf * scale);
```
