# 力控底盘技术文档

## 1. 概述

本文档描述了底盘从**速度控制(速控)**改为**力矩控制(力控)**的技术实现。力控模式相比速控具有更好的动态响应和打滑控制能力。

### 1.1 速控 vs 力控对比

| 特性 | 速控模式 | 力控模式 |
|------|---------|---------|
| 控制目标 | 轮子转速 | 轮子力矩/电流 |
| 闭环结构 | 电机内部速度环+电流环 | 应用层速度环+电机电流环 |
| 响应速度 | 较慢 (受速度环限制) | 较快 (直接控制力矩) |
| 打滑处理 | 被动 (速度偏差检测) | 主动 (力矩效率检测) |
| 调参难度 | 简单 | 中等 |

### 1.2 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层 (chassis.c)                        │
├─────────────────────────────────────────────────────────────────┤
│  目标速度 (vx, vy, wz)                                           │
│       ↓                                                         │
│  ┌─────────────────┐    ┌─────────────────┐                     │
│  │   速度外环PID    │ ←  │  卡尔曼滤波器   │ ← IMU + 轮速        │
```

## 2. 核心算法

### 2.1 速度融合 (卡尔曼滤波)

使用一维卡尔曼滤波器融合IMU加速度积分和轮速计算的速度：

```c
typedef struct {
    float x;      // 状态估计值 (速度)
    float P;      // 估计协方差
    float Q;      // 过程噪声 (IMU积分误差)
    float R;      // 测量噪声 (轮速误差)
} SimpleKalman1D_t;
```

**预测步骤** (基于IMU加速度):
$$x_{pred} = x + a \cdot dt$$
$$P_{pred} = P + Q$$

**更新步骤** (基于轮速):
$$K = \frac{P_{pred}}{P_{pred} + R}$$
$$x = x_{pred} + K \cdot (wheel\_vel - x_{pred})$$
$$P = (1 - K) \cdot P_{pred}$$

**力控模式参数**:
- `Q = 0.05` (IMU积分更可信)
- `R = 0.8` (轮速权重降低)

### 2.2 速度外环PID

力控模式下，速度环在应用层实现，输出为目标力：

```c
// 速度误差 → 目标力
Fx = PID(real_vx, target_vx)  // X方向力 (N)
Fy = PID(real_vy, target_vy)  // Y方向力 (N)
Mz = PID(real_wz, target_wz)  // 旋转力矩 (N·m)
```

### 2.3 逆运动学解算

X型全向轮布局：
```
   lf ／＼ rf
      X
   lb ＼／ rb
```

**底盘力 → 轮力分解**:
$$F_{lf} = \frac{-F_x - F_y}{\sqrt{2}} + \frac{M_z}{4L}$$
$$F_{rf} = \frac{F_x - F_y}{\sqrt{2}} - \frac{M_z}{4L}$$
$$F_{lb} = \frac{-F_x + F_y}{\sqrt{2}} - \frac{M_z}{4L}$$
$$F_{rb} = \frac{F_x + F_y}{\sqrt{2}} + \frac{M_z}{4L}$$

**轮力 → 电机电流**:
$$I = \frac{F \cdot R_{wheel}}{K_t \cdot Ratio}$$

其中:
- $R_{wheel}$ = 轮子半径
- $K_t$ = 电机力矩常数 (M3508 ≈ 0.3 N·m/A)
- $Ratio$ = 减速比

### 2.4 打滑检测 (力矩效率法)

**原理**: 打滑时，输出力矩不能有效转化为加速度

$$\eta = \frac{a_{actual}}{a_{expected}} = \frac{a_{IMU}}{F_{total}/m}$$

**检测逻辑**:
```c
if (torque_efficiency < 0.5f) {
    // 打滑: 实际加速度不到预期的50%
    is_slipping = 1;
    tcs_factor = max(torque_efficiency, 0.4f);
}
```

**TCS响应**:
- 打滑时降低力矩输出: `torque *= tcs_factor`
- 增大卡尔曼R值，降低轮速权重
- 渐进恢复，避免震荡

## 3. 代码结构

### 3.1 关键数据结构

```c
// 打滑控制结构体
typedef struct {
    float slip_ratio_x, slip_ratio_y;  // 滑移率 (备用)
    uint8_t is_slipping;               // 打滑标志
    float tcs_factor;                  // TCS输出系数 [0.4, 1.0]
    float wheel_vx, wheel_vy;          // 轮速计算的速度
    
    // 力控专用
    float expected_accel;    // 预期加速度 = F/m
    float actual_accel;      // 实际加速度 (IMU)
    float torque_efficiency; // 力矩效率
    float total_force_x, total_force_y;  // 总力
} SlipControl_t;
```

### 3.2 主要函数

| 函数 | 功能 |
|------|------|
| `ChassisInit()` | 初始化电机(电流环)、卡尔曼滤波器、速度外环PID |
| `EstimateSpeed()` | 速度融合 + 打滑检测 |
| `ForceCalculate()` | 速度外环PID + 逆运动学解算 |
| `LimitChassisOutput()` | 应用TCS + 设置电机参考值 |
| `ChassisTask()` | 主循环: 获取指令 → 速度估计 → 力解算 → 输出 |

### 3.3 控制流程

```
ChassisTask()
    │
    ├── 获取控制指令 (CAN/PubSub)
    │
    ├── 坐标变换 (云台系 → 底盘系)
    │
    ├── EstimateSpeed()
    │       ├── 轮速逆运动学
    │       ├── 卡尔曼滤波融合
    │       └── 打滑检测 (力矩效率)
    │
    ├── ForceCalculate()
    │       ├── 速度外环PID → Fx, Fy, Mz
    │       ├── 逆运动学 → 轮力
    │       └── 力矩→电流转换
    │
    └── LimitChassisOutput()
            ├── 应用TCS因子
            └── DJIMotorSetRef()
```


```c
// X/Y方向速度PID
```

## 5. 与速控的差异总结

| 项目 | 速控实现 | 力控实现 |
|------|---------|---------|
| 电机闭环 | `SPEED_LOOP \| CURRENT_LOOP` | `CURRENT_LOOP` |
| 速度环 | 电机模块内部 | 应用层 `ForceCalculate()` |
| 输出变量 | `vt_lf, vt_rf...` (轮速) | `torque_lf, torque_rf...` (力矩) |
| 解算函数 | `MecanumCalculate()` | `ForceCalculate()` |
| 打滑检测 | 滑移率 | 力矩效率 |
| 卡尔曼R | 0.5 (正常) / 5.0 (打滑) | 0.8 (正常) / 3.0 (打滑) |
